# 產品需求文件 (PRD) - 線上抽籤系統

## 1. 專案概述
- **背景與動機**: 許多活動（如尾牙、社團抽獎、行銷活動等）常有抽籤需求，傳統實體抽籤方式不僅耗時，還易引發公平性疑慮或名單遺漏的問題。本系統旨在提供一個數位化、透明且易於操作的線上抽籤平台，讓主辦單位能輕鬆管理抽獎活動並快速產出抽籤結果。
- **目標用戶**: 
  - **活動主辦單位/管理員**: 需要建立抽籤活動、匯入及管理名單，並在活動當下操作抽籤流程的負責人（如公司福委會、行銷人員等人）。
  - **一般參與者**: 想查詢自己是否中獎或查看過去抽籤紀錄的活動參與人。
- **核心價值主張**: 簡單易用的介面、公平且隨機的數位抽籤機制，以及便利的名單與結果管理。

## 2. 功能需求

### 1. 建立抽籤活動
- 作為 **管理員**，我希望 **能夠建立新的抽籤活動（包含名稱、抽籤獎項與活動說明）**，以便 **清楚區分不同的抽獎場次或活動**。

### 2. 參加者資料管理
- 作為 **管理員**，我希望 **能夠手動新增或批次匯入參加者名單**，以便 **快速準備抽獎名冊**。
- 作為 **管理員**，我希望 **能夠編輯或刪除已經加入的參加者**，以便 **在名單變動時保持資料的正確性**。

### 3. 執行抽籤
- 作為 **管理員**，我希望 **能夠按下抽籤按鈕，讓系統從現有參加者中隨機抽出指定得獎人數**，以便 **快速、公平地選出中獎者**。
- 作為 **管理員**，我希望 **能夠設定「不重複中獎」等規則**，以便 **讓每個人都只有一次中獎機會**。

### 4. 抽籤結果顯示與查詢
- 作為 **管理員/一般參與者**，我希望 **能在螢幕上清晰看到抽籤結果與動畫效果**，以便 **提升開獎的趣味性與公開透明度**。
- 作為 **一般參與者**，我希望 **能查詢過往所有已結束或已開獎的結果清單**，以便 **確認自己是否有中獎**。

### 5. 帳號登入與權限管理
- 作為 **管理員**，我希望 **能夠透過帳號密碼登入系統的後台介面**，以便 **保護活動資料不被一般參與者隨意修改**。

## 3. 非功能需求
- **技術限制**: 後端建議使用 Flask 框架，搭配 Jinja2 樣板引擎處理畫面，並且使用 SQLite 作為資料庫（根據專案標準技術棧）。
- **效能與安全考量**: 抽籤過程的隨機演算法需確保不可預測性；管理員的後台需要基本的驗證（Session / Cookie 防護）以防止越權存取與修改。

## 4. MVP 範圍

### Must Have (必須要有)
- 基本的系統管理員靜態登入（或單一管理員帳號）。
- 能新增一個抽籤活動，並可新增參與者。
- 點擊抽籤按鈕後，能在後端運算出隨機結果記錄到資料庫，並顯示。
- 顯示過往抽籤結果列表。

### Should Have (應該要有)
- 可批次匯入（如上傳 CSV ）並管理（修改/刪除）參加者名單。
- 抽到的結果可以提供給非管理員（一般人）透過特定連結查看，而無需登入。

### Nice to Have (有了更好)
- 以生動的抽籤動畫（如大轉盤或跑馬燈）呈現開獎過程。
- 多活動與多帳號角色權限管理系統。
